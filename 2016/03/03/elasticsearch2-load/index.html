Elasticsearch 2.1.1压力测试

发表于 2016-03-03   |   分类于 技术 ， elasticsearch   |  

测试环境

硬件配置

软件

  • elasticsearch-2.1.1 内存分配为4G
  • jmeter-2.10
  • zabbix-3.0

Index测试一

测试目的

  1. 索引的建立系统的瓶颈在哪个地方？
  2. 增加节点，索引速度是否符合线性增长？

索引数据量：1亿
Elasticsearch索引库结构：

        "search_analyzer": "ik_smart"
      },
      "properties" : {
        "dest_ip":{
          "type": "ip",
          "store": "no",
          "include_in_all": "true"
        },
        "date_time":{
          "type":"date",
          "format": "yyyy/MM/dd HH:mm:ss||yyyy/MM/dd||epoch_millis"
        },
        "dns":{
          "type":"string"
        },
        "source_ip":{
          "type":"ip"
        },
        "dns_server":{
          "type":"ip"

          "type":"string"

测试结论

  1. 批量建立索引的过程中，单台机器无副本的情况下CPU是性能的瓶颈。(通过监控看出CPU高负载主要表现在大量的上下文切换，并不表现在磁盘io等待。负载高导致ES无法响应)
  2. 通过测试对比得出，每增加一台机器，索引的性能基本符合线性递增(无副本的情况下)

Search测试一

测试目的

  1. 索引查询过程中，系统的瓶颈在哪里？
  2. 增加节点，查询速度是否符合线性增长？

查询条件

数据量总大小约：1亿条

{
  "from": 0,
  "size": 20,
  "query": {
    "filtered": {
      "query": {
        "bool": {
          "must": [
            {
              "match": {
                "dns": "${dns}"
              }
            }
          ]
        }
      },
      "filter": {
        "bool": {
          "must": [
            {
              "term": {
                "ttl": "${ttl}"
              }
            },
            {
              "range":{
                "source_ip":{
                  "gte":"${source_ip}",
                  "lte":"10.115.65.255"
                }
              }
            }
          ]          
        }
      }
    }
  }
}

5 shards 0 replicats 1 nodes

并发100个查询
平均完成一个查询耗用时间：2.93 秒
每秒完成的查询次数为： 34.1 次

查询系统资源占用截图：

5 shards 0 replicats 2 nodes

并发100个查询
平均完成一个查询耗用时间：2.49 秒
每秒完成的查询次数为： 40.0 次
【注：这里仅仅选择一台节点的数据进行显示，节点是配置相同，数据分布时也是均匀的】

查询系统资源占用截图：

测试结论

  1. 在并发查询过程中，系统的瓶颈同样在CPU
  2. 查询过程中，增加节点，查询的性能并没有线性增长，仅仅只是查询响应时间有所好转，查询的吞吐量增大

Search测试三

测试目的

  1. 主要测试ElasticSearch-2.1.1在大量的聚合查询下，是否会发生内存堆栈溢出？

查询条件

数据量大小约：1 亿

/bap2/dns/_search
{
  "query": {
    "filtered": {
      "query": {
        "match_all": {}
      },
      "filter": {
        "range": {
            "source_ip":{
              "gte":"${source_ip}",
              "lte":"10.115.65.255"
            }
        }
      }
    }
  },  
  "aggs": {
    "dns_aggs": {
      "terms": {
        "field": "dns",
        "size": 10,
        "order": {
          "_term": "asc"
        }        
     }
    }
  }
}

5 shards 0 replicats 2 nodes

并发20个查询线程
平均完成一个查询耗用时间：3.99秒
每秒完成的查询次数为： 5.0次

查询JVM资源占用截图：

测试结论

  1. 在es-2.1.1中，并发大量的聚合查询并未发现JVM堆栈溢出的情况
